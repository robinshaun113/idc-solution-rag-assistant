## 1. 百炼 embedding 报 400 `contents is neither str nor list of str`

- **现象**：调百炼 text-embedding-v2 报 400 InvalidParameter。
- **原因**：langchain-openai 默认把文本先用 tiktoken 切成 token 数组再发，但百炼 OpenAI 兼容接口只认字符串原文。
- **解法**：`OpenAIEmbeddings(check_embedding_ctx_length=False)`，关掉这个预处理。
- **面试一句话**：第三方 OpenAI 兼容接口和官方 SDK 有细微行为差异，要看实际请求体定位，而不是只看文档。

## 2. 改了 .env 的 Key 却不生效

- **现象**：修改 .env 里的 API Key，程序仍用旧值。
- **原因**：shell 里残留的同名环境变量优先级高于 .env 文件值，覆盖了文件。
- **解法**：`load_dotenv(..., override=True)`，强制用文件值覆盖环境变量。
- **面试一句话**：环境变量加载有优先级，配置不生效优先排查"是不是被更高优先级的源覆盖了"。

## 3. 真实 PDF 接入：扫描件 PyPDF 抽不出文字（2026/6/10）

- **现象**：接入 5 份真实 PDF 国标/白皮书后，`GBT51314数据中心基础设施运行维护标准.pdf`（18MB、43页）用 PyPDFLoader 抽取出 **0 字符**，其余 4 份正常（共约 19.4 万字符）。
- **原因**：该 PDF 是**扫描件（图片版）**，页面是图像而非文字层，PyPDF 这类文本抽取器抽不出内容。文件体积大（18M/43页 ≈ 0.4M/页）也是扫描件的典型信号。
- **解法（技术分层）**：先用 4 份文字版 PDF 跑通索引与评估；扫描件需要 OCR（光学字符识别）才能抽文字，归入 P3 多模态阶段用 PaddleOCR 统一处理，不在 P1 提前引入。
- **面试一句话**：接入真实文档时要先做"可解析性体检"——文字版直接抽、扫描件走 OCR，按文档类型做技术分层，而不是一个 loader 硬扛所有格式。
